# Текущая численная постановка

Этот документ описывает именно ту модель, которая сейчас реализована в
репозитории и запускается через `DOLFINx`.

## 1. Что считается сейчас

Решается `2D` скалярная задача рассеяния на круговой частице.

Геометрия:

- внутренняя круговая частица радиуса `a = 0.2`;
- внешняя круговая граница радиуса `R = 1.0`.

Это не `3D` сфера, а именно `2D` круг в сечении. Поэтому физически это ближе
к модели рассеяния на цилиндре, чем к полной теории Ми для сферы.

## 2. Геометрия и сетка

Геометрия задается в:

## 3. Неизвестная величина

Сейчас solver ищет не полное поле, а рассеянное поле `u_sc`.

После решения полное поле восстанавливается как:

```text
u_tot = u_inc + u_sc
```

Такой подход лучше подходит для scattering-задачи, чем прежняя схема с
жестким условием `u = u_inc` на всей внешней границе.

## 4. Уравнение в области

В области решается скалярная форма уравнения Гельмгольца для рассеянного поля:

```text
-Δu_sc - k0^2 n(x)^2 u_sc = f_inc
```

где:

- `k0` — волновое число в вакууме или в выбранной нормировке;
- `n(x)` — показатель преломления, кусочно-постоянный по подобластям;
- `f_inc` — объемный источник, который строится из падающего поля.

Слабая форма задается в:

- [scattering_2d.py](../ufl/scattering_2d.py)

## 5. Как устроен источник

Правая часть разбита на две части:

```text
f_inc = f_bg + k0^2 (n(x)^2 - n_bg^2) u_inc
```

Здесь:

- `u_inc` — заданное падающее поле;
- `n_bg` — показатель преломления фона;
- `f_bg = Δu_inc + k0^2 n_bg^2 u_inc`.

Смысл такой:

- вторая часть отвечает за рассеяние на материальном контрасте;
- первая часть нужна потому, что не всякое заданное `u_inc` является точным
  решением фонового уравнения Гельмгольца.

### Что это дает на практике

Для плоской волны:

```text
u_inc(x, y) = cos(k_bg x)
```

получается `f_bg = 0`, то есть источник сидит только в области контраста.

Для гауссова пучка:

```text
u_inc(x, y) = exp(-((y - y0) / w0)^2) cos(k_bg x)
```

фоновая часть `f_bg` уже не равна нулю, потому что такая функция не является
точным решением фонового уравнения Гельмгольца.

## 6. Граничное условие

На внешней границе используется однородное Robin-условие:

```text
∂n u_sc + alpha u_sc = 0
```

где `alpha > 0`.

В текущей реализации:

```text
alpha = k0 * n_background
```

Это не точное radiation condition, а real-valued поглощающий компромисс,
который помогает не задавать падающее поле жестко на внешней границе.

## 7. Материалы

Сейчас по умолчанию используются:

- `n_background = 1.0`
- `n_particle = 1.5`

То есть:

- в фоне `n^2 = 1.0`;
- в частице `n^2 = 2.25`.

Поле `n^2` сохраняется отдельно в:

- [material_n2_2d.pvd](../results/material_n2_2d.pvd)

## 8. Какие падающие поля реализованы

Источник задается в:

- [main.cpp](../src/main.cpp)

### 8.1. Плоская волна

Сейчас используется real-valued плоская волна вдоль оси `x`:

```text
u_inc(x, y) = cos(k_bg x)
```

где:

- `k_bg = k0 * n_background`.

### 8.2. Гауссов пучок

Сейчас используется real-valued модель:

```text
u_inc(x, y) = exp(-((y - y0) / w0)^2) cos(k_bg x)
```

где:

- `w0` — ширина пучка;
- `y0` — поперечное смещение центра пучка.

## 9. Какие поля сохраняются

Для каждого режима сейчас сохраняются:

- падающее поле `u_inc`;
- рассеянное поле `u_sc`;
- полное поле `u_tot = u_inc + u_sc`;
- фоновая часть источника `f_bg`.

## 10. Ограничения текущей модели

Важно честно держать в голове ограничения:

- это `2D` скалярная модель, а не полный `3D` Максвелл;
- расчет real-valued, а не complex-valued;
- Robin-условие на внешней границе только аппроксимирует поглощение;
- гауссов пучок задан как удобный аналитический профиль, а не как точное
  решение в бесконечной области;
- аналитическое сравнение пока не реализовано.

## 11. Почему это уже хорошая рабочая версия

Несмотря на упрощения, текущая модель уже дает:

- реальную геометрию частицы на сетке `gmsh`;
- материал частицы и фона;
- два разных режима возбуждения;
- раздельные поля `u_inc`, `u_sc`, `u_tot`;
- более честную scattering-постановку, чем простое навязывание `u = u_inc`
  на внешней границе.

То есть это уже хороший базовый численный прототип, на который можно
наращивать аналитику и более физичную постановку.

## 12. Что улучшать дальше

Следующие разумные шаги:

1. Перейти к комплексной постановке `Re/Im`, не полагаясь на complex-режим `PETSc`.
2. Сделать более физичное условие на внешней границе.
3. Добавить сравнение с аналитикой для `2D` цилиндра.
4. После этого думать про `3D`.
